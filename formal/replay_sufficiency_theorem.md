# Replay Sufficiency Theorem, Formal Version

This file gives a compact formal statement for the ReplayCapsule-RV target
theorem. It is deliberately modest: the theorem is conditional on the
assumptions in `formal/assumptions.md`.

## Objects

Let:

- `P` be a platform profile.
- `F` be an immutable firmware image.
- `A` be an architectural state.
- `E` be a finite, ordered boundary event stream.
- `Obs` be replay-visible observations, including MMIO writes, traps,
  exceptions, and safety-checker observations.
- `Trace(P, F, A0, E)` be the architectural trace generated by the deterministic
  replay semantics from initial state `A0` while consuming event stream `E`.

An event has the shape:

`e = (commit_index, order, kind, payload)`

Events are sorted lexicographically by `(commit_index, order)`.
In an implementation, `order` may be a global monotonic `seq`; the proof only
requires that events at the same commit index have a stable total order.

The trace is a finite sequence:

`T = [(A_0, o_0), (A_1, o_1), ..., (A_n, o_n)]`

where each transition is either:

- A firmware instruction retirement.
- A trap-entry transition.
- A recorded external architectural state transformation.

Each trace element carries enough metadata to recover the current
`commit_index`. Instruction retirement increments `commit_index`; trap-entry
and external asynchronous transitions do not.

## Replay Capsule

A replay capsule is:

`C = (image_id, platform_id, A0_or_reset, E_prefix, W_prefix, S_target, meta)`

where:

- `image_id` identifies `F`.
- `platform_id` identifies `P`.
- `A0_or_reset` gives `A0` directly or gives deterministic reset inputs that
  derive `A0`.
- `E_prefix` contains nondeterministic boundary events through the target point.
- `W_prefix` contains MMIO write observations used for validation.
- `S_target` is the target safety-property violation signature.
- `meta` contains hashes, recorder version, and other integrity data.

## Replay Equivalence

Two traces `T` and `T'` are replay-equivalent through target point `t`, written:

`T ~=_t T'`

when all of the following hold for the prefix through `t`:

- Corresponding architectural states are equal.
- Corresponding commit-index metadata are equal.
- Corresponding consumed boundary event payloads are equal.
- Corresponding replay-visible observations are equal.
- The safety checker emits the same violation signature at `t`, if any.

This equivalence abstracts away microarchitectural state and unobserved device
state.

## Theorem

Let `R` be an original run on `(P, F)` from initial state `A0`. Let `T_R` be its
architectural trace through target point `t`. Let `C` be a replay capsule for
`R` through `t`.

If:

1. `C.image_id` identifies the same `F`.
2. `C.platform_id` identifies the same `P`.
3. `C.A0_or_reset` yields the same initial architectural state `A0`.
4. `C.E_prefix` contains exactly the nondeterministic boundary event payloads
   that affect `R` through `t`, with the same `(commit_index, order)`.
5. `C.W_prefix` matches the MMIO write observations produced by `R` through
   `t`.
6. Assumptions A0 through A11 hold.

Then:

`Trace(P, F, A0, C.E_prefix) ~=_t T_R`

and if `T_R` produces target violation signature `S_target` at `t`, then the
replay trace also produces `S_target` at `t`.

## Proof Sketch

Proceed by induction over the ordered transition prefix through `t`.

Base case:

The capsule and assumptions give the same `P`, same `F`, and same initial
architectural state `A0`. Therefore the original and replay traces start in
equal states with equal commit-index metadata.

Inductive hypothesis:

Immediately before transition `j`, the original and replay states are equal,
their commit-index metadata are equal, and all prior observations are equal.

Case analysis for transition `j`:

1. Ordinary instruction retirement.
   The instruction fetch/decode inputs, registers, CSRs, privilege mode, and RAM
   bytes are equal. By deterministic core semantics, the resulting state and
   observations are equal.

2. MMIO read instruction.
   Equal pre-state gives the same load instruction, address, width, byte lanes,
   and fault context. Boundary completeness gives the same recorded read payload
   at the same commit point. Applying `step(P, F, A, input)` therefore produces
   equal next states and observations.

3. MMIO write instruction.
   Equal pre-state gives the same store address, width, byte lanes, data, and
   fault context. Replay validates this against `W_prefix`. The architectural
   core state after the store is deterministic. Any later device response must
   appear as a later event, so no hidden device state is needed for this step.

4. Interrupt boundary or trap entry.
   Equal pre-state includes equal CSR and privilege state. The capsule provides
   the same interrupt-line or delivery payload at the same `(commit_index,
   order)`. By the precise-interrupt assumption and deterministic arbitration,
   the same trap decision and trap-entry state result.

5. External architectural mutation.
   For DMA or another externally caused RAM/state change, event completeness
   gives the same payload at the same position. Applying the event produces the
   same state transformation in both traces.

These cases cover every transition allowed by the platform profile and event
model. Therefore the invariant is preserved through `t`.

Finally, the safety property checker is deterministic over the equal trace
prefix and equal observations. It emits the same target violation signature.

## Honest Limits

The theorem fails if any assumption fails. In particular:

- Missing MMIO reads, interrupt events, DMA writes, reset inputs, or observable
  timer/counter values can make replay diverge.
- Commit-index ordering is insufficient for properties that require physical
  cycle time unless cycle time is itself modeled as replay-visible input.
- Hidden device state is not reproduced unless all later architectural effects
  of that state are recorded.
- Multihart executions need a memory-interleaving model and event ordering
  stronger than the single-hart assumptions here.
- The theorem does not establish recorder correctness or instrumentation
  noninterference.
- The theorem only covers the finite prefix through the chosen violation.
