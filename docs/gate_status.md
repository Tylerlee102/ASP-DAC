# ReplayCapsule-RV Acceptance Gate Status

Last inspected: 2026-06-26.

## Scope and Evidence Rules

I did not find a canonical repository file that defines exactly ten acceptance
gates by name. The ten gates below are derived from `docs/research_plan.md`,
`docs/verification_plan.md`, `docs/baselines.md`, `docs/ablation_plan.md`,
`docs/experimental_methodology.md`, `artifact_evaluation.md`, and the current
source/results tree.

Status vocabulary:

- Complete: the gate's required repo artifacts are present and supported by
  current evidence.
- Partially complete: meaningful implementation or generated evidence exists,
  but the gate is not end-to-end complete.
- TODO: the repo contains a plan or directory, but not runnable implementation
  evidence for the gate.
- Blocked-by-missing-tool: the repo has an intended path, but local acceptance
  depends on a missing executable or toolchain.

Current local tool caveat:

- `python` and `python3` resolve to Windows Store aliases in this shell, but
  the checked-in `scripts/reproduce_all.ps1` wrapper finds the bundled Codex
  Python runtime and successfully runs the local gate.
- `make`, a C++ compiler entry point, and `riscv64-unknown-elf-gcc` are missing. System `yosys` is also
  missing, but the local gate can use workspace-local `yowasp-yosys` and
  `yowasp-sby`/`yowasp-yosys-smtbmc` from `.tools/python/`, plus OSS CAD Suite
  HDL tools from `.tools/oss-cad-suite/`, when installed outside git tracking.
- Existing files under `results/processed/` were regenerated by the local gate
  after the six-benchmark model expansion, Yosys synthesis pass, formal
  expansion, and directed-HDL verification expansion.
- The workspace is now a git repository with local checkpoint commits.

## Bottom Line

ReplayCapsule-RV is currently a serious scaffold with real six-benchmark
model-level and firmware-sim replay evidence, static RTL contract checks,
source-present RTL, firmware source pairs, RTL-smoke capsule export self/negative/PC-context checks, RTL/firmware-sim alignment rows,
generated figures, bounded SMTBMC replay-control, capsule-buffer, and recorder proof/cover checks, and measured generic Yosys cell counts.
It now has twelve firmware-running PicoRV32 wrapper smokes, but it is not yet an end-to-end
benchmark-wide RV32I RTL replay artifact, and it still has no mapped FPGA
resource, timing, or runtime-overhead results in this workspace.
