## Scope and Evidence Rules

This file tracks the ten top-conference acceptance gates from the original
ReplayCapsule-RV project request. It is not a replacement gate list.

Status vocabulary:

- Complete: the gate's named deliverable is present and supported by current
  evidence.
- Partially complete: meaningful generated evidence exists, but the gate is not
  satisfied at the full hardware/benchmark-wide scope implied by the project.
- TODO: the repo contains a plan or placeholder, but not runnable evidence.
- Blocked-by-missing-tool: the intended path depends on a missing local
  executable or toolchain.

Current local tool caveat:

- `python` and `python3` resolve to Windows Store aliases in this shell, but
  the checked-in `scripts/reproduce_all.ps1` wrapper finds the bundled Codex
  Python runtime for local script-based gates.
- The locked Linux CI run is the authority for compiler-backed firmware,
  Verilator full RTL replay, mapped ECP5 synthesis/place-and-route, paper
  build, audits, and artifact packaging.
- Existing files under `results/processed/` were regenerated by the local gate
  after the six-benchmark model expansion, firmware-sim path, RTL-smoke capsule
  export checks, RTL/firmware-sim alignment table, Yosys synthesis pass,
  directed HDL checks, and formal expansion.
- The workspace is a git repository with local checkpoint commits.

## Bottom Line

The canonical documentation, RTL, replay, mapped synthesis, reviewer,
reproducibility, and honesty gates are supported by the locked CI evidence. The
remaining limitations are scope limitations: host-driven replay consume,
single-hart RV32I interrupt/MMIO focus, no ASIC/power claim, and high prototype
area overhead.
