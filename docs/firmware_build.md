# Firmware Image Build

`scripts/build_firmware_images.py` generates deterministic RV32I firmware image artifacts for all six benchmark programs.

Output directory:

```text
firmware/build/
```

For each benchmark and each variant (`failing`, `fixed`), the script writes:

- `<variant>.hex`: one 32-bit instruction word per line.
- `<variant>.mem`: Verilog `$readmemh`-style address/data rows.
- `<variant>.json`: metadata including reset PC, labels, word count, sensor value, command value, and interrupt commit.

The images are generated by the same assembler used by `scripts/rv32i_firmware_sim.py`, so they are executable by the local RV32I interpreter and suitable as seed artifacts for a future RTL memory testbench.

Current limitation:

These images are not produced by an external RISC-V C compiler. They are hand-built benchmark programs encoded through the local RV32I assembler subset. A future RISC-V GCC/Clang flow should compile the C firmware sources and compare or replace these images.
