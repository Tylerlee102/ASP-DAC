# Experimental Methodology

The evaluation must be generated from scripts and raw artifacts only. No result table should be filled by hand.

## Workloads

Initial target benchmarks:

- sensor threshold response deadline
- interrupt race during MMIO update
- MMIO ordering violation
- stack/control-flow store corruption
- UART command unsafe branch
- watchdog heartbeat timeout

The current runnable path covers all six benchmark classes at two non-RTL evidence levels:

- `model`: event-boundary model from `scripts/replaycapsule_model.py`
- `firmware-sim`: RV32I instruction interpreter from `scripts/rv32i_firmware_sim.py`

Firmware-running RTL results currently include one PicoRV32 wrapper smoke for the sensor-threshold failing image. The full benchmark suite is pending a broader PicoRV32 simulation harness, `make`/C++ support for Verilator-style builds, and a RISC-V compiler or equivalent firmware image flow.

## Metrics

- replay success rate
- capsule size
- trace reduction ratio
- LUT/FF/BRAM overhead
- Fmax before/after
- runtime slowdown
- cycles to failure
- events per kilo-instruction
- buffer overflow rate
- false property failures
- missed property failures
- minimum event set per benchmark

## Honesty Policy

Unavailable measurements are `TODO` or `NA`. Yosys generic cell counts may be reported after a real Yosys run, but FPGA-mapped LUT/FF/BRAM/Fmax results require OpenROAD, vendor tools, or a board flow.

## Evidence Levels

- `model`: generated by the deterministic event-boundary model. Useful for replay semantics, baseline logic, and ablation coverage.
- `firmware-sim`: generated by the local RV32I instruction interpreter. Useful for proving that the benchmark programs execute as instruction words and emit ReplayCapsule evidence.
- `rtl-static`: generated by static SystemVerilog contract checks. Useful for module/event-interface coverage, not timing or synthesis.
- `rtl-frontend`: generated by directed Icarus simulations and Verilator lint-only checks. Useful for HDL frontend coverage and small directed module behavior.
- `rtl-smoke`: generated by the Icarus PicoRV32 wrapper smoke test. Useful for proving one generated firmware image can execute through the wrapper and trigger/freeze the expected property failure.
- `rtl-sim`: full benchmark-wide firmware-running PicoRV32 simulation remains pending.
- `synthesis`: generated by Yosys for generic cell counts when available; mapped FPGA resources and timing remain pending until a mapped flow exists.

## Current Generated Tables

- `results/processed/replay_experiments.csv`: six model-level commit-index replay rows.
- `results/processed/firmware_sim_replay.csv`: six RV32I firmware-sim commit-index replay rows.
- `results/processed/trace_sizes.csv`: model-level baseline sizes and replay success.
- `results/processed/ablations.csv`: model-level event removal, buffer-size, and last-K sweeps.
- `results/processed/event_sufficiency.csv`: per-benchmark event classes whose removal breaks replay.
- `results/processed/hdl_checks.csv`: directed Icarus simulations, one PicoRV32 wrapper smoke, and Verilator lint-only HDL frontend checks.
- `results/processed/synthesis.csv`: Yosys generic cell counts when available; honest TODO/NA fields for LUT/FF/BRAM/Fmax until a mapped FPGA flow exists.
