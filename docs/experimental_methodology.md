# Experimental Methodology

The evaluation must be generated from scripts and raw artifacts only. No result table should be filled by hand.

## Workloads

Initial target benchmarks:

- sensor threshold response deadline
- interrupt race during MMIO update
- MMIO ordering violation
- stack/control-flow store corruption
- UART command unsafe branch
- watchdog heartbeat timeout

The current runnable path covers all six benchmark classes at model, firmware-sim, smoke RTL, and Verilator full RTL evidence levels:

- `model`: event-boundary model from `scripts/replaycapsule_model.py`
- `firmware-sim`: RV32I instruction interpreter from `scripts/rv32i_firmware_sim.py`

Firmware-running RTL results include fifteen PicoRV32 wrapper smokes and 45 compiler-backed Verilator full RTL replay rows across the evaluated benchmark, variant, and seed cases. A generated smoke summary parses wrapper logs and checks observed property IDs, capsule counts, freeze state, and overflow state. A seeded RTL-smoke interrupt campaign reruns selected interrupt-race schedules in fresh Icarus processes, checks capsule digest reproducibility, sweeps MMIO wait-state latency for interrupt-race cases, and verifies per-seed replay-comparator rejection of missing-event, duplicate-event, metadata, payload, and order corruptions. A generated alignment table checks that failing RTL-smoke capsules agree with the RV32I firmware interpreter on property IDs and key replay-visible event values.

## Metrics

- replay success rate
- capsule size
- trace reduction ratio
- LUT/FF/BRAM overhead
- Fmax before/after
- runtime slowdown from measured cycle/commit and simulator wall-time summaries
- cycles to failure
- events per kilo-instruction
- buffer overflow rate
- false property failures
- missed property failures
- minimum event set per benchmark

## Honesty Policy

Unavailable measurements are `TODO` or `NA`. Yosys generic cell counts and generic cell deltas may be reported after real Yosys runs, but FPGA-mapped LUT/FF/BRAM/Fmax results require OpenROAD, vendor tools, or a board flow.

## Evidence Levels

- `model`: generated by the deterministic event-boundary model. Useful for replay semantics, baseline logic, and ablation coverage.
- `firmware-sim`: generated by the local RV32I instruction interpreter. Useful for proving that the benchmark programs execute as instruction words and emit ReplayCapsule evidence.
- `rtl-static`: generated by static SystemVerilog contract checks. Useful for module/event-interface coverage, not timing or synthesis.
- `rtl-frontend`: generated by directed Icarus simulations and Verilator lint-only checks. Useful for HDL frontend coverage and small directed module behavior.
- `rtl-smoke`: generated by Icarus PicoRV32 wrapper smoke tests. Useful for proving selected generated firmware images can execute through the wrapper, trigger/freeze expected property failures, and export parseable smoke-level capsules with memory-event PC-context sanity checks plus comparator rejection of missing-event, duplicate-event, metadata-corrupted, payload-corrupted, and order-corrupted capsules.
- `formal-bmc`: generated by bounded SMTBMC proof and cover harnesses over small RTL configurations. Useful for event routing, classifier/slicer capture policy, property-checker failure IDs/signatures, hash-signature updates, MMIO/interrupt logger state, register decode/control behavior, replay-control and replay-mismatch guards, recorder invariants, and reachability checks such as freeze, overflow, and event-count bounds, not for full processor proof.
- `rtl-sim`: generated by the Verilator full RTL record/replay harness.
- `synthesis`: generated by Yosys for generic cell counts, derived generic cell-overhead context, and same-target ECP5 mapped LUT/FF/BRAM/Fmax rows.

## Current Generated Tables

- `results/processed/replay_experiments.csv`: six model-level commit-index replay rows.
- `results/processed/replay_negative_tests.csv`: benchmark-derived replay-comparator positive and negative fixture checks across commit-index and cycle-index modes.
- `results/processed/firmware_sim_replay.csv`: six RV32I firmware-sim commit-index replay rows.
- `results/processed/rtl_capsule_exports.csv`: twelve failing/fixed RTL-smoke capsule export parse/self-compare, missing-event, duplicate-event, metadata-corruption, payload-corruption, order-corruption, and memory-event PC-context rows.
- `results/processed/rtl_capsule_event_classes.csv`: event-class counts and packet bytes decoded from exported RTL-smoke capsules.
- `results/processed/rtl_smoke_ablations.csv`: event-class removal ablations over exported RTL-smoke capsules using the replay comparator.
- `results/processed/rtl_smoke_event_sufficiency.csv`: required RTL-smoke event classes derived from those exported-capsule ablations.
- `results/processed/rtl_firmware_alignment.csv`: twelve RTL-smoke versus firmware-sim property and key-event alignment rows.
- `results/processed/randomized_interrupt_campaign.csv`: seeded RTL-smoke interrupt-race reruns with capsule-digest reproducibility checks and MMIO wait-state latency cases.
- `results/processed/randomized_interrupt_corruption.csv`: per-seed replay-comparator rejection checks for corrupted seeded RTL-smoke capsules.
- `results/processed/trace_sizes.csv`: model-level and firmware-sim baseline sizes and replay success, plus RTL-smoke capsule-byte rows where exported packets exist.
- `results/processed/ablations.csv`: model-level event removal, buffer-size, and last-K sweeps.
- `results/processed/event_sufficiency.csv`: per-benchmark event classes whose removal breaks replay.
- `results/processed/hdl_checks.csv`: ten directed Icarus module simulations, fifteen PicoRV32 wrapper smokes, and Verilator lint-only HDL frontend checks.
- `results/processed/picorv32_smoke_summary.csv` and `results/processed/picorv32_smoke_coverage.csv`: log-level wrapper smoke checks for property IDs, capsule counts, freeze state, overflow state, failing-image capsules, fixed-image no-failure capsules, and no-failure edge capsules.
- `results/processed/benchmark_coverage.csv`: six-benchmark local coverage ledger joining model replay, firmware-sim replay, wrapper smokes, RTL-smoke exports, alignment, and sufficiency rows.
- `results/processed/full_rtl_replay.csv`: compiler-backed Verilator full RTL record/replay rows.
- `results/processed/full_rtl_replay_negative.csv`: full RTL replay-critical corruption-rejection rows.
- `results/processed/formal_checks.csv`: bounded SMTBMC event-tap, event-classifier/slicer, property-checker, hash-signature, MMIO/interrupt logger, registers, replay-control, replay-mismatch, capsule-buffer, and recorder proof/cover checks.
- `results/processed/overflow_contracts.csv`: directed and bounded-formal overflow contract evidence plus an explicit TODO row for benchmark-wide runtime overflow rate.
- `results/processed/synthesis.csv`: Yosys generic cell counts.
- `results/processed/synthesis_overhead.csv`: derived generic cell-overhead context between baseline PicoRV32 and the integrated wrapper.
- `results/processed/mapped_synthesis.csv`, `results/processed/mapped_overhead.csv`, `results/processed/mapped_recorder_presence.csv`, and `results/processed/full_core_mapped_summary.csv`: same-target ECP5 board-level mapped evidence and recorder presence checks.
- `results/processed/evaluation_metrics.csv`: headline metric rollup from generated replay, trace-size, RTL-smoke, full RTL, runtime, and synthesis artifacts, with unsupported out-of-scope metrics left as `TODO` or `NA`.
- `results/processed/claim_audit.csv`: generated audit of high-risk claim phrases, with uncaveated rows treated as local gate failures.
- `results/processed/toolchain_status.csv`: generated tool availability ledger showing which local tools are present and which missing tools block later gates.
- `results/processed/proof_obligations.csv`: conservative mapping from theorem assumptions to current generated evidence and remaining limits.
- `paper/figures/table01_synthesis_resources.md`: generated Markdown paper table source derived from the synthesis CSVs.
- `paper/figures/table02_replay_evidence.md` through `paper/figures/table07_evaluation_metrics.md`: generated Markdown paper table sources for replay evidence, trace-size baselines, event-sufficiency ablations, bounded formal coverage, replay-sufficiency proof obligations, and evaluation metrics.
