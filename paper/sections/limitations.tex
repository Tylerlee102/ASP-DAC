\section{Limitations}

The scope is single-hart RV32I firmware with deterministic core semantics, the same program image, the same initial architectural state, and modeled external events. The current theorem excludes multicore interleavings, DMA unless modeled, cache/coherence effects unless modeled, analog nondeterminism, and claims about undefined C behavior.

MMIO side effects are modeled only at the replay-visible boundary. Interrupts are injected at an architectural boundary. Replay consumes every required event exactly once; missing, duplicate, reordered, or corrupted events must fail comparison. Capsule overflow invalidates replay availability.

The replay-consume path is host-driven in the Verilator harness. The mapped design measures the recorder/instrumented full-core wrapper, not a synthesizable autonomous replay-consume engine. High FF/LUT overhead remains an optimization target. ASIC power/area/timing is not evaluated unless a real ASIC flow is added. The evaluation is single-hart RV32I and does not cover multicore, DMA, caches/coherence, or arbitrary peripherals.

ReplayCapsule-RV v2 reduces the architectural risk by adding compressed recorder RTL and a synthesizable replay-consume controller prototype, but the prototype is not fully integrated into the full-core replay harness. The v2 workload, buffer, capsule-size, and mapped-overhead tables therefore distinguish measured v1 evidence from analytic v2 estimates and blocked full-core v2 rows. These rows should not be used to claim measured v2 replay success or measured full-core v2 overhead improvement.
