\begin{abstract}
Embedded firmware failures often depend on interrupt timing and memory-mapped I/O values that are visible to software but absent from ordinary crash reports. ReplayCapsule-RV records commit-indexed capsules for a scoped class of single-hart RV32I interrupt/MMIO bugs and replays those boundary events in an RTL harness. The implementation integrates a record-side hardware capsule path with PicoRV32, compiler-built RV32I firmware images, Verilator record/replay execution, corruption checking, runtime summaries, and same-target FPGA mapping. The generated artifact reports \rcFullReplayPass/\rcFullReplayTotal\ compiler-backed full RTL replay PASS rows and rejects \rcNegativeReject\ replay-critical corrupted capsules with \rcNegativeUnexpectedAccept\ unexpected accepts. The current prototype validates event sufficiency and replay reproducibility for bring-up/debug instrumentation; it does not claim an autonomous full-core hardware replay engine or area-optimized shipped silicon.
\end{abstract}
