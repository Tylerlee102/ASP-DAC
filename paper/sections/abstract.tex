\begin{abstract}
Embedded firmware failures often depend on interrupt timing and memory-mapped I/O values that are visible to software but absent from ordinary crash reports. ReplayCapsule-RV records commit-indexed capsules for a scoped class of single-hart RV32I interrupt/MMIO bugs and replays those boundary events in an RTL harness. The implementation integrates a record-side hardware capsule path with PicoRV32, compiler-built RV32I firmware images, Verilator record/replay execution, an RTL capsule capture/source path, a replay-mode controller, an RTL replay consumer that drives core-facing MMIO/IRQ replay, corruption checking, runtime summaries, same-target FPGA mapping, and OpenROAD ASIC/open-PDK evidence. The generated artifact reports \rcFullReplayPass/\rcFullReplayTotal\ v1 and \rcVTwoReplayPass/\rcVTwoReplayTotal\ v2 compiler-backed full RTL replay PASS rows, \rcExpandedReplayPass/\rcExpandedReplayTotal\ expanded benchmark rows across \rcExpandedBenchmarks\ additional firmware families, \rcSelfReplayPass/\rcSelfReplayTotal\ captured-store self-replay handoff rows, a 42/42 focused RTL self-replay shell matrix, and a 48/48 seeded Hazard3 MMIO+IRQ replay matrix. The current prototype validates event sufficiency and replay reproducibility for bring-up/debug instrumentation with controller-driven captured-store replay evidence; it does not claim a standalone board/silicon replay engine or area-optimized shipped silicon.
\end{abstract}
